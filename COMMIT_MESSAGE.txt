Fix: Guest data และ payment status inconsistency

## ปัญหาที่แก้ไข

1. Payment status ไม่ตรงกันระหว่าง admin/checkin และ admin/reception
   - admin/checkin แสดง "ยังไม่ชำระ" 
   - admin/reception แสดง "ยืนยันแล้ว"

2. Guest data ใช้ mock data แทนข้อมูลจาก guest account
   - แสดง "Fon Testuser, fon.test@example.com, 0867890006"
   - แทนที่จะใช้ข้อมูลจริงจาก signed-in guest account

## การแก้ไข

### 1. Payment Status Logic (booking_repository.go)
- เพิ่ม CASE statement ใน GetArrivals query
- ถ้า booking status = Confirmed/CheckedIn/Completed → payment_status = "approved"
- เพราะ Confirmed หมายความว่าผ่าน confirm_booking() แล้ว (ชำระเงินสำเร็จ)

### 2. Guest Data Logic (booking_service.go)
- แก้ CreateBooking ให้ใช้ guest account data สำหรับ signed-in users
- ใช้ข้อมูลจาก guest account เสมอ (ไม่ใช้ข้อมูลจากฟอร์ม)
- เพิ่ม logging เพื่อ debug

### 3. Fix Existing Bookings (fix_mock_guest_data.sql)
- SQL script สำหรับ update booking_guests table
- แก้ไข booking เก่าที่มี mock data ให้ใช้ข้อมูลจาก guest account
