# Task 28: Verification Checklist - Receptionist Check-in/out Interface

## ภาพรวมการทดสอบ

เอกสารนี้ใช้สำหรับตรวจสอบว่า Task 28 ทำงานถูกต้องครบถ้วนตามความต้องการ

## สิ่งที่ต้องเตรียม

### 1. ข้อมูลทดสอบ
- [ ] มีบัญชี Receptionist สำหรับทดสอบ
- [ ] มีการจองที่ยืนยันแล้วสำหรับวันนี้
- [ ] มีห้องว่างที่สะอาดหรือตรวจสอบแล้ว
- [ ] มีแขกที่เข้าพักอยู่สำหรับทดสอบ check-out
- [ ] มีการจองที่เกินกำหนด check-in สำหรับทดสอบ no-show

### 2. สภาพแวดล้อม
- [ ] Backend API ทำงานอยู่ (http://localhost:8080)
- [ ] Frontend ทำงานอยู่ (http://localhost:3000)
- [ ] Database มีข้อมูลทดสอบ
- [ ] เบราว์เซอร์ที่รองรับ (Chrome, Firefox, Safari, Edge)

- [ ] คลิกปุ่มได้
- [ ] แสดง loading state (กำลังเช็คอิน...)

### 3. Error Handling
